## üé® **Implementation Patterns**

### **‚úÖ 1. Dual Accessibility Approach**
```tsx
// Pattern: aria-label + sr-only for maximum compatibility
<Button
  aria-label="Descriptive action text"
  onClick={handleAction}
>
  <IconComponent className="h-4 w-4" />
  <span className="sr-only">Descriptive action text</span>
</Button>
```

**Benefits:**
- ‚úÖ **aria-label**: Provides accessible name for the button
- ‚úÖ **sr-only**: Ensures screen readers have text content to announce
- ‚úÖ **Redundancy**: Multiple accessibility methods for better compatibility

### **‚úÖ 2. Contextual Labels**
```tsx
// Pattern: Include context in labels for clarity
<Button aria-label={`Action ${itemName}`}>
  <Icon className="h-4 w-4" />
  <span className="sr-only">Action {itemName}</span>
</Button>
```

**Benefits:**
- ‚úÖ **Specific Context**: Users know exactly what will be affected
- ‚úÖ **Clear Purpose**: No ambiguity about button function
- ‚úÖ **Better UX**: Reduces cognitive load for screen reader users

### **‚úÖ 3. Responsive Icon Buttons**
```tsx
// Pattern: Text hidden on mobile but accessibility maintained
<Button aria-label="Full descriptive text">
  <Icon className="h-4 w-4" />
  <span className="hidden sm:inline">Visible Text</span>
  <span className="sr-only">Full descriptive text</span>
</Button>
```

**Benefits:**
- ‚úÖ **Mobile Optimization**: Saves space on small screens
- ‚úÖ **Desktop Clarity**: Shows text labels on larger screens
- ‚úÖ **Consistent Accessibility**: Screen readers get full context regardless of screen size

### **‚úÖ 4. State-Dependent Labels**
```tsx
// Pattern: Dynamic labels based on current state
<Button
  aria-label={isVisible ? "Hide content" : "Show content"}
  onClick={toggleVisibility}
>
  {isVisible ? <EyeOff className="h-4 w-4" /> : <Eye className="h-4 w-4" />}
  <span className="sr-only">
    {isVisible ? "Hide content" : "Show content"}
  </span>
</Button>
```

**Benefits:**
- ‚úÖ **Current State**: Users understand the current state
- ‚úÖ **Action Clarity**: Clear about what the button will do
- ‚úÖ **Dynamic Updates**: Labels update with state changes
